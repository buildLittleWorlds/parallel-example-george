# Parallel Example: Health Reading-Level Lab

This packet models how a health-explainer Space could become a research journal and paper-lite draft. It is adjacent to health explanation work, but it does not describe the student's actual project.

## Core idea

The imagined Space is a **Health Reading-Level Lab**. A user enters a medical topic and chooses an audience level. The Space rewrites the explanation for different readers, then shows what changed.

## Research question

When a health explainer changes its reading level, is it actually changing the explanation, or mostly swapping vocabulary and sentence length?

## Space idea

Inputs:

- Health topic or short medical paragraph
- Audience: elementary student, middle school student, adult patient, or pre-med student
- Optional mode: shorter, more examples, or more caution

Outputs:

- Rewritten explanation
- Readability estimate
- List of concepts kept, removed, or simplified
- Reflection prompt asking whether the simpler version lost meaning

## What this packet demonstrates

- How to turn "make this easier to understand" into a researchable question
- How to distinguish readability from accuracy and completeness
- How to use medical-education sources without claiming clinical authority
- How to make a health topic student-safe by focusing on communication

## How to use this example

Read `sources.md`, then the journal, then the paper. The example is designed to show how a student can make one concrete comparison instead of writing a broad essay on AI in medicine.
